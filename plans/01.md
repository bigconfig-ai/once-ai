# Plan: port the Once AI GitHub secret plugin

## Goal

Port `/Users/amiorin/code/personal/once-ai/src/io/github/bigconfig_ai/once_ai/plugin.clj` into this `once-website` package so a successful deployment updates the GitHub Actions org secret that carries the server IP.

## Current findings

- This repo is a concrete `once` package instance; deployment changes normally live in `run`.
- The source plugin updates org `bigconfig-ai`, secret `SERVER_IP`, using `gh`.
- It hooks after the `ansible-local` package step and reads `:ip` from `::workflow/params`.
- The source step keyword is old: `:io.github.amiorin.once.tools/ansible-local`; this package should use `:io.github.bigconfig-ai.once.tools/ansible-local`.
- The source uses `defmethod handle-step`; current BigConfig may use either a multimethod or the newer `register-handle-step` registry, so the port should be API-compatible.
- Guard against `package build`: the hook must not update `SERVER_IP` after a render-only build with fallback IP `192.168.0.1`.

## Implementation plan

1. Create a local namespace at `src/io/github/bigconfig_ai/once_website/plugin.clj` with namespace `io.github.bigconfig-ai.once-website.plugin`.
2. Add `"src"` to local paths in both `deps.edn` and `bb.edn` so Babashka can load the namespace.
3. Require the plugin namespace from `run` for its registration side effect, keeping the deployment profile in `run`.
4. In the plugin namespace, require:
   - `babashka.process`
   - `big-config`
   - `big-config.core`
   - `big-config.pluggable`
   - `big-config.workflow`
   - `cheshire.core`
   - `clojure.string`
5. Define private constants:
   - GitHub org: `bigconfig-ai`
   - secret name: `SERVER_IP`
6. Register a handler for `:io.github.bigconfig-ai.once.tools/ansible-local` with a compatibility helper:
   - if `big-config.pluggable/register-handle-step` exists, use it;
   - otherwise, if `pluggable/handle-step` is a `clojure.lang.MultiFn`, add a method;
   - fail early for unsupported APIs.
7. In the handler:
   - run the original `ansible-local` step first;
   - only continue if `::big-config/exit` is `0`;
   - only update the secret when the step actually ran `ansible-playbook main.yml` (create path), not when it only rendered (build path);
   - require a real IP and reject fallback/blank IPs.
8. Preserve the current org-secret visibility:
   - read `/orgs/<org>/actions/secrets/<secret-name>`;
   - if visibility is `selected`, fetch selected repos and pass `--repos` back to `gh secret set`;
   - if the secret does not already exist, default to `--visibility all`, matching the original plugin.
9. Verification:
   - run `./run package build` and confirm it does **not** call `gh secret set`;
   - run a local/safe handler smoke test with a fake `gh` in `PATH` or a temporary test secret;
   - after confirmation, run `./run package create` with `gh auth status` already valid.

## Decisions

- Plugin will live in a new namespace, not inline in `run`.
- Target stays org `bigconfig-ai`, secret `SERVER_IP`.
- Missing secret defaults to `--visibility all`, matching the original plugin.
- `gh` failures should fail the deployment, matching the original plugin.
- Secret update happens after successful `ansible-local`, matching the original plugin.
- `package delete` does not clear/remove the secret, matching the original plugin.
