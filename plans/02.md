# Plan 02 — gh org secret SERVER_IP after `bb ai create`

## Goal
After `bb ai create` finishes, push the provisioned host's IP to GitHub as the
`SERVER_IP` org-level Actions secret on `bigconfig-ai`, using the `gh` CLI.

## Plugin contents

- Function `update-github-secret [opts]`:
  - Reads `ip` from `(get-in opts [::workflow/params :ip])`.
  - **Inspect the existing secret to preserve its scope.** Calling
    `gh secret set` without `--visibility` silently resets the scope, and
    when visibility is `selected` you must also re-pass the repo list or
    the secret becomes inaccessible to its current consumers.

    1. `gh api /orgs/bigconfig-ai/actions/secrets/SERVER_IP` →
       `{visibility, selected_repositories_url, ...}`. Capture
       `visibility`.
    2. If `visibility` is `selected`, also call
       `gh api /orgs/bigconfig-ai/actions/secrets/SERVER_IP/repositories`
       and capture the list of repo IDs (`.repositories[].id`).
    3. If the secret doesn't exist (404), treat as first run: set
       `visibility = "all"` and skip the repos fetch.

  - Shell out via `babashka.process/shell` (throws on non-zero exit,
    which fails the workflow):
    - Always: `gh secret set SERVER_IP --org bigconfig-ai --body <ip>
      --visibility <visibility>`.
    - When `visibility = "selected"`: also pass
      `--repos <comma-joined-ids>` to keep the same repo set.

  - Returns `(merge opts (core/ok))`.

- `defmethod handle-step :io.github.amiorin.once.tools/ansible-local
   [f _step _step-fns opts]`:
  - Calls `(f opts)` to run upstream's `ansible-local` step.
  - On success (`(zero? (::bc/exit opts'))`) calls `update-github-secret`.
  - On failure, returns the failed opts unchanged — skip the gh update so a
    stale IP isn't published.
  - `ansible-local` is the last step in `once`'s `create` pipeline
    (`io.github.amiorin.once.package/create`), so this fires after the whole
    `bb ai create` workflow completes.

## Notes

- No new step keyword is introduced. The gh logic is a plain function called
